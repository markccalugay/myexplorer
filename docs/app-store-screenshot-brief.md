# MyExplorer App Store Screenshot Brief

This brief turns ButterKit's design-science guidance into an 8-screen storyboard for `MyExplorer`.

Core rules carried over from the ButterKit guide:

- Lead with the user's problem in the first 2 screenshots.
- Treat product features as the solution.
- Include social proof early, but only use real quotes, awards, or coverage with permission.
- Keep each screenshot focused on 1 idea.
- Use a tight visual system: few colors, few text styles, lots of breathing room.

## Format

- iOS: `2736 x 1260 px`
- Android: `1080 x 1920 px`

## Shared Visual System

- Palette: deep ocean `#0F3D56`, warm sand `#F3E7D3`, coral accent `#F26B5B`, leaf accent `#6E8F62`, white `#FFFFFF`
- Typography styles:
  - Title: bold, high contrast, short
  - Subtitle: one supporting sentence
  - Caption: optional small trust or feature label
- Layout rule: one hero device per screen, one supporting UI moment, large negative space, no more than one accent color per frame
- Photography rule: use authentic destination imagery or real in-app UI only; avoid generic AI-looking composites

## Storyboard

### 1. Problem: Trip planning is scattered

- Headline: `Travel planning is all over the place.`
- Subtitle: `Hotels, routes, activities, and notes usually live in different apps.`
- UI focus: home or explore view with destination discovery cards and a calm map presence
- Visual direction: show a single iPhone or Android device framed by soft destination imagery, with lots of empty space around the headline
- Goal: make the pain feel familiar before selling features

### 2. Problem: Group trips get messy fast

- Headline: `Group trips turn chaotic fast.`
- Subtitle: `Coordinating stops, vehicles, and plans shouldn't happen in scattered chats.`
- UI focus: convoy or trip-planner state that hints at shared planning
- Visual direction: use a cleaner, slightly more structured composition than screen 1; keep the mood reassuring, not stressful
- Goal: define the second major pain point for families and friend groups

### 3. Solution: Build the whole trip in one place

- Headline: `Plan the whole route in one flow.`
- Subtitle: `Map your origin, stops, destination, and timing without switching tools.`
- UI focus: `TripPlanner` with timeline, stops, and route context
- Visual direction: this is the first strong "hero" feature shot; let the planner UI occupy more space
- Goal: establish MyExplorer as the answer to screens 1 and 2

### 4. Solution: Discover places worth the drive

- Headline: `Find stays, activities, and hidden gems.`
- Subtitle: `Explore local-first recommendations, not just the usual chains.`
- UI focus: discovery or explore surfaces with lodging, activities, and local stories
- Visual direction: use destination imagery more heavily here, but keep the text short and the palette restrained
- Goal: position discovery as curated and distinctive

### 5. Solution: Route recommendations keep the trip moving

- Headline: `Make smarter stops along the way.`
- Subtitle: `Surface pitstops, essentials, dining, and detours right on your route.`
- UI focus: route recommendations, pitstops, and along-the-way planning
- Visual direction: emphasize motion and progress, with one route line or wayfinding cue supporting the device
- Goal: show practical trip intelligence, not just inspiration

### 6. Solution: Save places and start from where you are

- Headline: `Reuse favorites and start faster.`
- Subtitle: `Save home, pickup points, and must-visit places for one-tap planning.`
- UI focus: favorites, current-location origin, quick presets
- Visual direction: calmer composition with fewer UI elements; this screen should feel efficient and personal
- Goal: show convenience and repeat-use value

### 7. Solution + Social Proof: Coordinate the whole crew

- Headline: `Keep everyone on the same plan.`
- Subtitle: `Manage vehicles, assignments, and shared trip details together.`
- UI focus: convoy management or saved-trip coordination
- Visual direction: add a trust cue early on this screen only if it is real
- Trust cue options:
  - `Planned with MyExplorer`
  - a real customer quote with permission
  - a real usage number
- Important: do not fabricate reviews, download counts, or press logos
- Goal: blend feature clarity with trust

### 8. Outcome: Save trips and pick back up anytime

- Headline: `Your next getaway stays ready.`
- Subtitle: `Save trips, reopen them later, and head out with confidence.`
- UI focus: `Bookings` or saved-trips view
- Visual direction: most polished and reassuring frame in the set; end on clarity, not density
- Goal: leave the viewer with a sense of readiness and momentum

## Platform Layout Notes

### iOS `2736 x 1260`

- Use a landscape composition with the device slightly off-center and text block on the opposite side.
- Keep generous left and right margins so the frame feels premium instead of crowded.
- Let map or destination imagery stretch wider behind the device, but keep it soft and secondary.

### Android `1080 x 1920`

- Use a portrait stack: headline at top, device in the middle, short subtitle below or above depending on balance.
- Keep vertical spacing generous; do not cram the screen just because it is taller.
- Favor one large device render instead of multiple floating UI cards.

## ButterKit Build Notes

- Keep screenshot 1 and screenshot 2 problem-led.
- Put the strongest planner UI on screenshot 3.
- If you have a verified quote, use it on screenshot 4 or 7. If not, skip social proof rather than invent it.
- Reuse the same background system, device angle, and text styles across all 8 screens.
- Trim any screen whose subtitle feels longer than one breath.
- If a frame feels crowded, remove elements before shrinking them.
